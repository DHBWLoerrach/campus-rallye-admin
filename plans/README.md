## Findings considered and rejected

- A Postgres RPC wrapping all question writes in one transaction was not
  included. The existing question actions already use sequential writes and a
  compensating delete for failed creation. Reworking all question persistence
  into RPCs is a separate cross-cutting consistency project and would expand
  this feature beyond its intended scope.
- Database constraints for coordinate ranges and positive radii were not
  included. Server-side Zod validation will protect writes through this admin
  application; hardening direct database writes should be handled in a
  separate schema migration if desired.
- Address search/geocoding was not included. It introduces a second external
  service and its own usage policy; map click, marker drag, and exact coordinate
  inputs cover the first complete version.
